Niniejszy poradnik stanowi uzupełnienie poradnika: Jak zmienić skład w scenerii, z którym powinieneś się zapoznać najpierw (między innymi po to, aby wiedzieć co robić z wpisami).


LOKOMOTYWY, EZT:

node -1 0 EU07-150 dynamic PKP\4E_v1 4E-150.tga 4E 0.0 headdriver 3 0 enddynamic

  • -1 - Maksymalna odległość z jakiej widoczny jest model pojazdu (w metrach); -1 to nieskończoność, co w praktyce oznacza 2000 m; W celu niewielkiego polepszenia FPS, można zmniejszyć tę liczbę np. do 800.
  • 0 - Minimalna odległość z jakiej widoczny jest model pojazdu.
  • EU07-150 - Nazwa lokomotywy, która będzie wyświetlana w Rainsted.exe; w przypadku lokomotyw powinna być ona unikalna (w lokomotywach wielosekcyjnych i EZT tyczy się to każdego członu) w obrębie danej scenerii; natomiast nazwy wagonów mogą się powtarzać.
  • dynamic - Typ obiektu. Dynamic w uproszczeniu określa obiekt poruszający się po drogach i torach.
  • PKP\4E_v1 - Ścieżka dostępu do pojazdu w katalogu Dynamic. By zmienić lokomotywę, wystarczy zamiast 4E_v1, wpisać nazwę folderu z inną lokomotywą. Np. ET22_v1. Innym katalogiem głównym (alternatywą dla PKP) jest np. road, gdzie znajdują się pojazdy drogowe, których wpisy są analogiczne do wpisów pojazdów szynowych
  • 4E-150.tga - Nazwa pliku tekstury wraz z rozszerzeniem. Jeśli model nie posiada tekstur wymiennych wpisujemy none.
    Informacja dla użytkowników zaawansowanych: Jeśli nie podamy rozszerzenia, to symulator będzie próbował wczytać teksturę o rozszerzeniu domyślnym. Dla exe 735 kB (exe z Dizelpack'a) i starszych jest to rozszerzenie bmp, natomiast dla exe z PC2009 i nowszych możemy zdefiniować domyślne rozszerzenie (ma ono wpływ na cały symulator, nie tylko na wpisy taboru!) w pliku eu07.ini. Odpowiada za to wpis defaultext xxx (na przykład: gdy wpiszemy tga zamiast xxx, domyślnym rozszerzeniem stanie się tga).
  • 4E - Wspólna nazwa pliku *.chk i *.mmd pojazdu, który umieszczamy w scenerii (pliki te znajdują się w katalogu Dynamic/PKP/4E_v1).
  • headdriver - obsada:
    • headdriver - prowadzenie z przedniej kabiny
    • reardriver - prowadzenie z tylnej kabiny
    • nobody - bez obsługi. Stosujemy w członach B lokomotyw dwusekcyjnych (ET40, ET41, ET42), przy ukrotnieniu dwóch lokomotyw (np. 2 x EU07) w drugiej z nich i we wszystkich członach EZT, za wyjątkiem pierwszego (w nim używamy headdriver)
    • connected - ukrotnienie.
  • 3 - liczba definiująca rodzaj połączenia z następnym pojazdem (tzw. sprzęg).
    Mamy do dyspozycji następujące rodzaje "sprzęgów":
    • 0 - oznacza brak fizycznego połączenia (stosujemy, gdy nasz pojazd jest ostatni w składzie);
    • 1 - sprzęg śrubowy;
    • 2 - przewód główny (czerwony wężyk);
    • 4 - przewody sterowania wielokrotnego;
    • 8 - przewód wysokiego napięcia (WN);
    • 16 - mostek przejściowy;
    • 32 - przewód zasilający (żółty wężyk).


    Różne kombinacje otrzymujemy dodając do siebie liczby odpowiadające poszczególnym elementom. I tak na przykład:
    • 0 (stosujemy, gdy nasz pojazd jest ostatni w składzie);
    • 3 = sprzęg śrubowy (1) + przewód główny (2) - najbardziej powszechny; stosowany do połączeń między wagonami towarowymi, osobowymi, a także między lokomotywą, a wagonami;
    • 35 = sprzęg śrubowy (1) + przewód główny (2) + przewód zasilający (32) - stosowany do połączeń między wagonami w pociągach o podwyższonym standardzie (EIC itp.);
    • 55 = sprzęg śrubowy (1) + przewód główny (2) + przewód zasilający (32) + ukrotnienie (4) + mostek przejściowy (16);
    • 63 = sprzęg śrubowy (1) + przewód główny (2) + przewód zasilający (32) + ukrotnienie (4) + przewód WN (8) + mostek przejściowy (16) - stosowany w członie A lokomotyw dwusekcyjnych (ET40, ET41, ET42), przy ukrotnieniu dwóch lokomotyw (tylko w pierwszej z nich), a także pomiędzy poszczególnymi członami EZT.

Przykładowe wpisy.

Dla lokomotywy dwusekcyjnej:
Kod:
node -1 0 ET41-101-A dynamic PKP\ET41_v1 ET41-101-A.tga 203E-A 0.0 headdriver 63 0 enddynamic
node -1 0 ET41-101-B dynamic PKP\ET41_v1 ET41-101-B.tga 203E-B 0.0 nobody 3 0 enddynamic
Dla EZT:
Kod:
node -1 0 EN57-1012ra dynamic PKP\EN57_v1 EN57-1012ra.tga 6baII 0.0 headdriver 63 0 enddynamic
node -1 0 EN57-1012s dynamic PKP\EN57_v1 EN57-1012s.tga 6bsII 0.0 nobody 63 0 enddynamic
node -1 0 EN57-1012rb dynamic PKP\EN57_v1 EN57-1012rb.tga 6bbII 0.0 nobody 0 0 enddynamic
Dla dwóch EZT:
Kod:

node -1 0 EN57-1012s dynamic PKP\EN57_v1 EN57-1012s.tga 6bsII 0.0 nobody 63 0 enddynamic

WAGONY TOWAROWE:

node -1 0 5051-503320-2 dynamic PKP\Eaos_v1 401W-48.tga 401W 0.0 nobody 3 40 coal enddynamic

  • -1 - tak samo jak w przypadku lokomotyw.
  • 0 - tak samo jak w przypadku lokomotyw.
  • 5051-503320-2 - dowolna nazwa. Polecam nazywać np. Wagon1, Wagon2, Wagon3 itd., by wyłapać na których ewentualnie wagonach się symulator wysypuje. Nazwy wagonów w scenerii nie muszą być unikalne.
  • dynamic - jak w przypadku lokomotyw.
  • PKP\Eaos_v1 - ścieżka dostępu. W tym przypadku Eaos_v1. Dla innych wagonów, np. cystern: Zaes_v1.
  • 401W-48.tga - tak samo jak w przypadku lokomotyw.
  • 401W - tak samo jak w przypadku lokomotyw.
  • nobody - w przypadku wagonu zostawiamy nobody.
  • 3 - tak samo jak w przypadku lokomotyw.
  • 40 - ilość ładunku. O sposobie liczenia ładunku (w tonach lub na sztuki [pasażerowie w wagonach pasażerskich]) w tym wagonie informuje plik 401W.chk. Istotny jest wpis LoadQ - pieces oznacza sztuki, natomiast tonns - tony.
  • coal - rodzaj ładunku. O rodzaju ładunku, który możemy wykorzystać w tym wagonie informuje również 401W.chk; Istotny jest wpis LoadAccepted. Dla tego wagonu możemy wybrać jeden z kilku dostępnych rodzajów (Coal, Stone, Wood, Lumber, Sand, Gravel) lub pozostawić wagon bez ładunku. Wtedy w miejscu ilość ładunku wpisujemy 0 i NIE WPISUJEMY JEGO RODZAJU. W przeciwnym wypadku przy ładowaniu symulatora na pewno wystąpi błąd.

Przykładowe wpisy.
Dla pustej cysterny:
Kod:
node -1 0 5051-503320-2 dynamic PKP\Zaes_v1 406Ra_dec8.tga 406Ra 0.0 nobody 3 0 enddynamic
Dla cysterny wypełnionej paliwem:
Kod:
node -1 0 5051-503320-2 dynamic PKP\Zaes_v1 406Ra_dec8.tga 406Ra 0.0 nobody 3 49 Fuel enddynamic

WAGONY PASAŻERSKIE:
Analogicznie jak dla wagonów towarowych. Różnica dotyczy ładunku - najczęściej stosujemy pasażerów (Passengers). Ładunek ten jest liczony w sztukach.
