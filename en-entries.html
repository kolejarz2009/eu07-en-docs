LOCOMOTIVES, EMUs:

node -1 0 EU07-150 dynamic PKP\4E_v1 4E-150.tga 4E 0.0 headdriver 3 0 enddynamic

  • -1 - maximal distance from which the vehicle is visible (in meters); -1 is infinity , which in practice, means 2000m; To slightly improve the FPS, this can be decreased to, for example 800.
  • 0 - Minimal distance from which the vehicle is visble.
  • EU07-150 - Displayed vehicle name - Shown in the Rainsted.exe; in case of locomotives this should be unique (applies to every section in multi-sectioned locos and in every car of Multiple Units) within this scenerio; carraige names can be repeated.
  • dynamic - Object type. Dynamic simply means anything moving on the rails and the roads.
  • PKP\4E_v1 - Folder path to the vehicle in the Dynamic folder. To change a locomotive type, simply replace the 4E_v1 with a folder of another locomotive type. For example, ET22_v1. Another main folder (alternative for the PKP) is for example road, which has all the road vehicles, whose entries are same as those for rail vehicles
  • 4E-150.tga - Name of the texture file along with it's file extension. If the model has no replaceable textures, type none here.
    Advanced User Information: If we don't type the extension then the Sim will try to load the texture with the default file extension. For exe of 735 kB in size (exe from the Dizelpack) and older the default extension is .bmp, while the newer exe from the PC2009 and newer, we can manually choose the default extension (it affects the WHOLE Simulator, not just the vehicle entries!) in the eu07.ini file. Use the defaultext xxx entry for that (for example if we type tga instead of xxx, the tga becomes the default).
  • 4E - Common name of the .chk / .fiz and the .mmd files for the vehicle which we place into the scenerio (in this case those files are located in the dynamic/pkp/4e_v1).
  • headdriver - occupancy:
    • headdriver - driving from the front cab
    • reardriver - driving from the rear cab
    • nobody - nobody occupies it. Used in the B section of multisectioned locos (ET40, ET41, ET42), when running two locos in multiple (for example 2 x EU07) in the second one and in all cars of the Multiple Units, except for the first one (use headddriver then)
    • connected - running in multiple.
  • 3 - number defining the type of conncetions with the next vehicle (the so-called coupling).
    We can choose the following types of "couplings":
    • 0 - no physical connection (use when this is the last vehicle in the set);
    • 1 - screw coupling;
    • 2 - brake pipe (red pipe);
    • 4 - multiple running connector;
    • 8 - High Voltage cable;
    • 16 - gangway;
    • 32 - main reservoir pipe (yellow pipe).


    We get many combinations adding the coresponding connection numbers. So for example:
    • 0 (use when our vehicle is last in the set);
    • 3 = screw coupling (1) + brake pipe (2) - most common; used to connect freight cars, passenger carriages and the loco to the first carriage;
    • 35 = screw coupling (1) + brake pipe (2) + main reservoir pipe (32) - used to connect the carriages in higher standard trains (Polish EIC etc.);
    • 55 = screw coupling (1) + brake pipe (2) + main reservoir pipe (32) + multiple running connector (4) + gangway (16);
    • 63 = screw coupling (1) + brake pipe (2) + main reservoir pipe (32) + multiple running connector (4) + High Voltage cable (8) + gangway (16) - used in the first section (A) of multi-sectioned locos (ET40, ET41, ET42), when running locos in multiple (only in the first one) and inbetween each car of the EMUs.

Example entries.

For locomotive with two sections:
Code:
node -1 0 ET41-101-A dynamic PKP\ET41_v1 ET41-101-A.tga 203E-A 0.0 headdriver 63 0 enddynamic
node -1 0 ET41-101-B dynamic PKP\ET41_v1 ET41-101-B.tga 203E-B 0.0 nobody 3 0 enddynamic
For EMUs:
Code:
node -1 0 EN57-1012ra dynamic PKP\EN57_v1 EN57-1012ra.tga 6baII 0.0 headdriver 63 0 enddynamic
node -1 0 EN57-1012s dynamic PKP\EN57_v1 EN57-1012s.tga 6bsII 0.0 nobody 63 0 enddynamic
node -1 0 EN57-1012rb dynamic PKP\EN57_v1 EN57-1012rb.tga 6bbII 0.0 nobody 0 0 enddynamic
For two EMUs:
Code:
node -1 0 EN57-1012ra dynamic PKP\EN57_v1 EN57-1012ra.tga 6baII 0.0 headdriver 63 0 enddynamic
node -1 0 EN57-1012s dynamic PKP\EN57_v1 EN57-1012s.tga 6bsII 0.0 nobody 63 0 enddynamic
node -1 0 EN57-1012rb dynamic PKP\EN57_v1 EN57-1012rb.tga 6bbII 0.0 nobody 63 0 enddynamic
node -1 0 EN57-1051ra dynamic PKP\EN57_v1 EN57-1051ra.tga 6baII 0.0 nobody 63 0 enddynamic
node -1 0 EN57-1051s dynamic PKP\EN57_v1 EN57-1051s.tga 6bsII 0.0 nobody 63 0 enddynamic
node -1 0 EN57-1051rb dynamic PKP\EN57_v1 EN57-1051rb.tga 6bbII 0.0 nobody 0 0 enddynamic

FREIGHT CARS:

node -1 0 5051-503320-2 dynamic PKP\Eaos_v1 401W-48.tga 401W 0.0 nobody 3 40 coal enddynamic

  • -1 - same as with the locos.
  • 0 - same as with the locos.
  • 5051-503320-2 - any name. We recommend using for example. Car1, Car2, Car3 etc., to catch on which carriage the Sim crashes (if it does). Carriage names don't have to be unique.
  • dynamic - same as with the locos.
  • PKP\Eaos_v1 - folder path to the vehicle. In this case it's Eaos_v1. For other cars, for example oil tankers: Zaes_v1.
  • 401W-48.tga - same as with the locos.
  • 401W - same as with the locos.
  • nobody - in the case of a carriage, we leave it as a nobody.
  • 3 - same as with the locos.
  • 40 - amount of the load. For this car type, the 401W.chk file informs us of the load counting method (in tonnes or it's quantity [passengers in the passenger carriages]). The LoadQ is the important line - pieces means individual items (including passengers), tonns however, means tonnes of the load (ie coal, oil etc)
  • coal - type of the load. The 401W.chk file also informs us of the load type; The important entry is the LoadAccepted. For this car, we can choose on of the few available loads (Coal, Stone, Wood, Lumber, Sand, Gravel) or leave the car ladden (empty). In that case, the quantity section is 0 and NOTHING for it's type. Otherwise the Sim will crash on start-up.

Example entries.
For an empty tanker:
Code:
node -1 0 5051-503320-2 dynamic PKP\Zaes_v1 406Ra_dec8.tga 406Ra 0.0 nobody 3 0 enddynamic
For a tanker filled with fuel:
Code:
node -1 0 5051-503320-2 dynamic PKP\Zaes_v1 406Ra_dec8.tga 406Ra 0.0 nobody 3 49 Fuel enddynamic

PASSENGER CARRIAGES:
Just like the freight cars, only difference is the load - usually Passengers. However, the load is counted in it's quantity, not mass.

Example entries
For a second class carriage:
Code:
node -1 0 5051-503320-2 dynamic PKP\1xxa_v1 111A13.tga 1xxa 0 nobody 3 0 enddynamic
For second class carriage with 30 passengers:
Code:
node -1 0 5051-503320-2 dynamic PKP\1xxa_v1 111A13.tga 1xxa 0 nobody 3 30 Passengers enddynamic
