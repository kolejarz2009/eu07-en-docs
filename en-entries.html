The following manual is an extension to the: How to change the train set in the scenery manual, with which you should familiarise yourself with first (So that you'd know what to do with the entries, etc). Read the whole thing and then start!!!


LOCOMOTIVES:
node -1 0 EU07-150 dynamic PKP\4E_v1 4E-150 4E 0.0 headdriver 3 0 enddynamic

WAGONS, CARRIAGES, EMUs and DMUs:
node -1 0 SN61-168 dynamic pkp/sn61_v2 sn61-168.dds sn61 0.0 headdriver 63 40 passengers enddynamic

  • -1 - Maksymalna odległość z jakiej widoczny jest model pojazdu (w metrach); -1 to nieskończoność, co w praktyce oznacza 2000 m; W celu niewielkiego polepszenia FPS, można zmniejszyć tę liczbę np. do 800.
  • 0 - Minimalna odległość z jakiej widoczny jest model pojazdu.
  • SN61-168 - Nazwa lokomotywy, która będzie wyświetlana w Rainsted.exe; w przypadku lokomotyw powinna być ona unikalna (w lokomotywach wielosekcyjnych i EZT tyczy się to każdego członu) w obrębie danej scenerii; natomiast nazwy wagonów mogą się powtarzać.
  • dynamic - Typ obiektu. Dynamic w uproszczeniu określa obiekt poruszający się po drogach i torach.
  • pkp/sn61_v2 - Ścieżka dostępu do pojazdu w katalogu Dynamic. By zmienić pojazd, wystarczy zamiast sn61_v2, wpisać nazwę folderu z inną lokomotywą. Np. et22_v2. Innym katalogiem głównym (alternatywą dla pkp) jest np. road, gdzie znajdują się pojazdy drogowe, których wpisy są analogiczne do wpisów pojazdów szynowych

    • Starymi standardami było podawanie ścieżek, używając Windowsowego ukośnika wstecznego, oraz używanie małych i dużych liter Teraz należy używać zwykłego ukośnika oraz małych liter. Tzn., było PKP\SN61_V2, a teraz ma być pkp/sn61_v2

  • SN61-168.dds - Nazwa pliku tekstury wraz z rozszerzeniem. Jeśli model nie posiada tekstur wymiennych wpisujemy none.
    Informacja dla użytkowników zaawansowanych: Jeśli nie podamy rozszerzenia, to symulator będzie próbował wczytać teksturę o rozszerzeniu domyślnym. Dla exe 735 kB (exe z Dizelpack'a) i starszych jest to rozszerzenie bmp, natomiast dla exe z PC2009 i nowszych możemy zdefiniować domyślne rozszerzenie (ma ono wpływ na cały symulator, nie tylko na wpisy taboru!) w pliku eu07.ini. Odpowiada za to wpis defaultext xxx (na przykład: gdy wpiszemy tga zamiast xxx, domyślnym rozszerzeniem stanie się tga).
  • SN61 - Wspólna nazwa pliku *.fiz i *.mmd pojazdu, który umieszczamy w scenerii (pliki te znajdują się w katalogu Dynamic/PKP/SN61_v2).
  • 0.0 - Wpis na razie nie używany
  • headdriver - obsada:
    • headdriver - prowadzenie z przedniej kabiny
    • reardriver - prowadzenie z tylnej kabiny
    • nobody - bez obsługi. Stosujemy w członach B lokomotyw dwusekcyjnych (ET40, ET41, ET42), przy ukrotnieniu dwóch lokomotyw (np. 2 x EU07) w drugiej z nich i we wszystkich członach EZT, za wyjątkiem pierwszego (w nim używamy headdriver)
    • 63 - Tak zwany "sprzęg". Suma poniższych cyfr, oznaczająca typ sprzęgów, połączonych z następnym pojazdem. Dodatkowo można dodać różne modyfikatory, dodając kropkę i modyfikatory, na końcu sprzęgu. Tzn.: 3.BR
      • 0 - oznacza brak fizycznego połączenia (stosujemy, gdy nasz pojazd jest ostatni w składzie);
      • 1 - sprzęg śrubowy;
      • 2 - przewód główny (czerwony wężyk);
      • 4 - przewody sterowania wielokrotnego;
      • 8 - przewód wysokiego napięcia (WN);
      • 16 - mostek przejściowy;
      • 32 - przewód zasilający (żółty wężyk).

      • Modyfikatory: (można dodawać razem, tzn.: 63.BP.WF9P100H99.TA)
        • W - Wpisy dotyczące kół
          • FxRyPz - Wielkość podkucia:
            x to gwarantowane podkucie, w mm
            y to losowe zwiększenie podkucia o zakres 0 - y mm (wpis opcjonalny)
            z - prawdopodobieństwo podkucia, w procentach. (wpis opcjonalny)
          • Hxx - Szansa na wężykowanie tylko danego pojazdu, xx w zakresie od 0 do 100
        • B Ustawienia hamulca
          • R - Nastawa na pospieszny (Rapid)
          • P - Nastawa na osobowy (Passenger)
          • G - Nastawa na towarowy (Goods)
          • O - Hamulce odłączone
          • Q - Na starcie bez powietrza w Z.G.
        • T Ustawienia Temperatury
          • A - Wymusza by temperatura elementów lokomotywy była temperaturą otoczenia (~15'C)

    • 40 - ilość ładunku. Dla lokomotyw ustawić na 0 O sposobie liczenia ładunku (w tonach lub na sztuki [pasażerowie w wagonach pasażerskich]) w tym wagonie informuje plik sn61.fiz. Istotny jest wpis LoadQ - pieces oznacza sztuki, natomiast tonns - tony.
    • passengers - rodzaj ładunku. O rodzaju ładunku, który możemy wykorzystać w tym wagonie informuje również sn61.fiz; Istotny jest wpis LoadAccepted. Np dla wagonów na materiały sypkie, typu 401W, możemy wybrać jeden z kilku dostępnych rodzajów (Coal, Stone, Wood, Lumber, Sand, Gravel) lub pozostawić wagon bez ładunku. Wtedy w miejscu ilość ładunku wpisujemy 0 i NIE WPISUJEMY JEGO RODZAJU. W przeciwnym wypadku przy ładowaniu symulatora na pewno wystąpi błąd.

    Przykładowe wpisy.

    Dla pustej cysterny z podkuciem 3mm:
    Kod:
    node -1 0 5051-503320-2 dynamic pkp/zaes_v1 406Ra_dec8.tga 406Ra 0.0 nobody 3.WF3 0 enddynamic
    Dla cysterny wypełnionej paliwem, z nastawą towarową, i 100 szansą na wężykowanie:
    Kod:
    node -1 0 5051-503320-2 dynamic pkp/zaes_v1 406Ra_dec8.tga 406Ra 0.0 nobody 3.BGWH100 49 Fuel enddynamic

    Dla wagonu klasy drugiej z nastawą pospieszną:
    Kod:
    node -1 0 5051-503320-2 dynamic PKP\1xxa_v1 111A13.tga 1xxa 0 nobody 3.BR 0 enddynamic
    Dla wagonu klasy drugiej z 30 pasażerami:
    Kod:
    node -1 0 5051-503320-2 dynamic PKP\1xxa_v1 111A13.tga 1xxa 0 nobody 3 30 Passengers enddynamic
    Dla składu wagonów mieszanych:
    Kod:
    node -1 0 ET42-005-A dynamic pkp/et42_v2 112E-005-A-HIST.DDS 112E-A 0 headdriver 63.BPWF17H100 0 enddynamic
    node -1 0 ET42-005-B dynamic pkp/et42_v2 112E-005-B-HIST.DDS 112E-B 0 connected 35.BPWH100 0 enddymanic
    node -1 0 p1 dynamic pkp/11xa_v2 BDNU-51512071081-6.DDS 111AW-SPOT 0 nobody 35.BP 40 passengers enddynamic
    node -1 0 p2 dynamic pkp/11xa_v2 BDNU-51512071081-6.DDS 111AW-SPOT 0 nobody 3.BP 0 enddynamic
    node -1 0 t1 dynamic pkp/408w_v1 5316830-7.DDS 408W 0 nobody 0.BPWF10R10P15 10 coal enddynamic

    Powyższy skład mieszany, ma wszystkie pojazdy ustawione na nastawę osobową, z ukrotnionym czapajewem na przodzie. Pierwszy człon czapajewa ma podkucia na 17mm, a oba człony mają szanse na wężykowanie na 100%. Pierwszy wagon, ma 40 pasażerów, a drugi jest tlenowozem (pusty). Węglarka wiezie 10T węgla, i może mieć podkucie od 10mm do 20mm, z szansą na bycie podkutą na 15%.
